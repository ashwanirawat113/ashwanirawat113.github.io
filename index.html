Project Overview

Developed a complete virtual CAN bus simulation in Python to test error frame handling without physical hardware — covering three error types as defined by the CAN protocol.

  • Virtual CAN bus with loopback using python-can library (receive_own_messages=True)
  • Bit Error: Randomly flips LSB of data byte with 30% probability using XOR
  • Stuff Error: Detects 6+ consecutive identical bits violating bit-stuffing rules
  • CRC Error: Simulates checksum mismatch via fake CRC computation (sum % 256 vs 0xAA)
  • Custom listener classes log errors with timestamps; all results verified against expected CAN protocol behaviour
